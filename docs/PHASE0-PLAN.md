### P2. `/pnl` 페이지 (정적 JSON, hand-edited)

**Phase 0 Architecture:**
- `src/data/pnl-snapshot.json` — repo 안 정적 파일. 매월 본인이 직접 수정 commit.
- 데이터 sanitization 책임 = 본인. raw 매출·자산 절대 파일에 적지 않음.
- ISR 또는 force-dynamic 무관 — 정적 import.

**Schema:**
```ts
type PnlSnapshot = {
  month: string                // "2026-04"
  delta_pct_vs_prev_month: number   // % delta (전월 대비). 음수 가능.
  agent_contribution_pct: {        // sum=100. percentage only.
    [persona_slug: string]: number
  }
  activity_counts: {
    posts_published: number
    trades_executed: number
    leads_captured: number
  }
  last_updated: string         // ISO date
}
```

**No raw revenue values. No source API integration. No cron.** 매월 본인 manual update.

**Page:**
- `/pnl` — 단순 page. 위 schema 시각화 (텍스트 + 막대 그래프 inline SVG 또는 단순 div).
- Hero 위젯 (apex /): 한 줄 — "agents N개 운영, 이번달 활동 +X%". import 동일 JSON.

**Stale detection:** `last_updated` > 35일 전이면:
- apex hero 위젯 = **hide**.
- `/pnl` 페이지 = **banner only** (상단 "데이터 업데이트 필요"). data 본문 정상 렌더.

**Cron / multi-source / KIS·impact.com 통합 = 명시 Phase 1 DEFERRED.** Phase 0에서 절대 시도 안 함.
